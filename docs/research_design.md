# Functional Requirements

## Project Objective

The project must produce actionable insight into how organizational uncertainty reshapes employment stability and household resilience.

The final outcome should allow a policymaker, workforce strategist, or economic researcher to:

1. Identify periods of elevated labor volatility.
2. Assess whether financial volatility and labor volatility move together.
3. Evaluate whether labor volatility precedes measurable household financial strain.
4. Understand where risk appears to concentrate within the system.

---

## Required Analytical Outputs

### 1. Labor Volatility Dashboard

The system must produce a dashboard or visual interface that displays:

- Quarterly layoff totals (California tech)
- Number of firms conducting layoffs
- Geographic distribution (county-level optional)
- Trend over time (2022ñ2024)

The dashboard must clearly identify volatility spikes.

---

### 2. Financial vs Labor Volatility Comparison

The system must:

- Display rolling financial volatility metrics
- Display labor volatility metrics
- Allow visual comparison across time
- Calculate and display the Labor Shock Absorber Index (LSAI)

The interface must make clear whether volatility appears more concentrated in employment or financial performance.

---

### 3. Household Financial Stress Overlay

The system must:

- Display household credit stress indicators
- Allow comparison with layoff activity (including 1-quarter lag view)
- Highlight temporal alignment between labor shocks and household strain

---

### 4. Policy Interpretation Layer

The output must support structured interpretation, including:

- Identification of volatility transmission patterns
- Discussion of potential institutional buffers (e.g., unemployment insurance, retraining)
- Explicit acknowledgment of limitations and non-causal nature

---

## Success Criteria

The project is considered successful if it:

- Produces a clean, interpretable quarterly panel dataset
- Enables a clear visual comparison between labor volatility and household stress
- Demonstrates whether volatility concentration shifts over time
- Provides policy-relevant framing without making unsupported causal claims

---

## Non-Goals

This project does not aim to:

- Prove causation
- Attribute intent to firms
- Evaluate individual-level outcomes
- Conduct predictive modeling in the current phase

---

## Deliverables

- Public GitHub repository
- Reproducible scripts
- Data dictionary
- Policy-facing visual dashboard
- Brief summary memo outlining findings and implications
